NAME

dummy-webcam - Simulates a webcam that can then be used in video conferencing software.

SYNOPSIS

dummy-webcam setup <webcam|video|audio>
dummy-webcam play [FILE...]
dummy-webcam help <html|man|raw>
dummy-webcam

DESCRIPTION

dummy-webcam is a Bash script which takes advantage of v4l2loopback, ffmpeg and PulseAudio to substitute a webcam stream with any file supported by ffmpeg.

Without any parameter, dummy-webcam gives displays this help.

With the dummy-webcam setup form, you setup the virtual devices as needed. You need to use this form only once before playing any stream.

With the dummy-webcam play form, you will be able to send media files (video+audio, video only, audio only, still images or animated images) to the dummy webcam.

Notes:

•

dummy-webcam setup webcam combines the effect of dummy-webcam setup video and dummy-webcam setup audio so you don’t need to call these last two commands if you already ran the first.

•

The video virtual device to use in video conferencing applications will be identified as Dummy video device (0x0000).

•

The audio virtual device to use in video conferencing applications will be identified as Dummy microphone.

OPTIONS

dummy-webcam setup

webcam

Setup the video and audio virtual devices.

video

Setup only the video virtual device.

audio

Setup only the audio virtual device.

dummy-webcam play
FILE...

Any file that can be played by ffmpeg (mpeg, mp3, mp4, mkv, ogg, gif, png, jpeg...). You may specify more than one file in which case the files will be played one after the other in the specified order.

Note: if a thumbnail is associated with an mp3, dummy-webcam will show it in the video stream.

dummy-webcam help

html

Write the help in HTML on the standard output.

man

Write the help in man format on the standard output.

raw

Write the help source code on the standard output.

REQUIREMENTS

In order to use dummy-webcam, the following packages need to be installed:

•

v4l2loopback

•

PulseAudio

•

ffmpeg

On Debian-like distributions, these can be installed with the following command:

sudo apt install v4l2loopback ffmpeg

Notes:

•

dummy-webcam setup video requires you to have root (sudo) permissions, so don’t be surprised if it asks for your password.

•

dummy-webcam setup audio uses the module-null-sink and module-remap-source PulseAudio modules. module-null-sink is used to create virtual sinks and module-remap-source is used to create virtual sources that video conferencing application can identify as a microphone source.
